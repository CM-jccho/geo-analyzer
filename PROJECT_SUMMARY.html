1. 개요

Obsidian 볼트에 기획만 있던 GEO-Analyzer를 실제 동작하는 Next.js 웹 서비스로 구현·배포하고, 같은 앱에 AutoSec Scanner(보안 취약점 스캐너)를 추가했다.

제품한 줄 정의AI 연동라이브
GEO AnalyzerURL을 넣으면 AI(ChatGPT·Claude 등)가 그 사이트를 어떻게 보는지 진단Claude 실제 호출동작
AutoSec ScannerURL/GitHub 저장소의 보안 취약점(OWASP Top 10) 진단미연동 (Trivy/ZAP)프로덕션 mock

3. AutoSec Scanner /security

  • 입력 — URL 스캔(DAST) / GitHub 저장소(SAST) 탭
  • 엔진 — mock(기본/프로덕션) 또는 real: Trivy(SAST) / OWASP ZAP(DAST)
  • 대시보드(다크) — 심각도 요약 카드 + Recharts 도넛 + 취약점 리스트(위치·설명·해결방안)
  • 공유 — /scan/[id] 리포트 페이지 + 링크 복사
  • 플랜별 1일 한도 — Free 3 / Pro 50 / Enterprise 1000회 (x-plan-key 헤더)

현재 AI 미연동. 취약점 탐지는 전용 엔진이 정석이며, 해석·우선순위·수정코드 생성에 Claude 레이어를 추가하는 것이 후속 과제(아래 A-2).

7. 후속 마무리 작업

A. 곧바로 가능 (코드 준비됨 / API 호출)

  • Vercel KV 연동 — KV(Upstash) 생성·연결 → 결과/스캔 공유 링크 영속 + 정확한 레이트리밋. (코드 완료, 프로비저닝만 필요)
  • 보안 스캐너 AI 분석 레이어 — Claude로 취약점 우선순위화·맞춤 수정코드·자연어 리포트. Vercel에서 바로 동작.
  • 리드/문의 알림 — 접수 시 Telegram/Slack 실시간 알림(/api/contact, /api/lead의 TODO).

B. 별도 인프라 필요

  • 보안 스캐너 실제 엔진 워커 — Docker(Trivy + ZAP), SCANNER_MODE=real. (Vercel 서버리스는 엔진 실행 불가)
  • 결제 연동 — Toss Payments/Stripe로 Pro 구독·적용 대행 결제.
  • 실제 인증 — NextAuth/Clerk + DB. x-plan-key를 실제 계정·구독으로 교체.

C. 운영 · 품질

  • 커스텀 도메인 — Vercel Domains에 보유 도메인 연결.
  • 분석 비용 통제 — Free 진단 LLM 캐싱/2종 제한으로 API 비용 관리.
  • 공유 영속화 일관화 — GEO 결과·스캔 리포트 모두 KV 기반 통일.

D. 보안 점검 (중요)

진행 중 채팅에 노출된 자격증명은 모두 폐기 후 재발급 권장:

  • Vercel 토큰 — vercel.com/account/tokens → Revoke
  • Gmail 앱 비밀번호 — myaccount.google.com/apppasswords → 삭제·재발급 → Vercel SMTP_PASS 갱신
